# 第1部 はじめに

## 1. このマニュアルについて

このマニュアルは、株式会社マツケンの建設派遣スタッフ配置管理システム（以下「本システム」）の機能と操作方法をまとめたものです。

- **対象**：本社管理者・経営層を中心に、営業所長・事務担当者までカバーします。
- **前提知識**：Webブラウザ（Google Chrome、Microsoft Edge など）が使える方。ITの専門知識は不要です。
- **表記のきまり**：
  - 画面名・ボタン名・メニュー名は『カレンダー』のように『』で囲みます。
  - 実際に打ち込む文字は `admin` のようにバックティックで囲みます。
  - 入力必須の項目には「必須」、任意の項目には「任意」と明記します。
  - 注意が必要な箇所は「> 注意」、便利な豆知識は「> 💡ポイント」として記載します。

## 2. システム全体像

本システムは、建設現場へ派遣するスタッフの配置スケジュール管理と、それに関わる事務作業のデジタル化を目的としたWebアプリです。

### できることの全体像

```
[マスター登録] → [毎日の配置計画] → [現場での作業確認] → [支払い計算] → [印刷・出力]
    営業所          カレンダー         出来高確認書        日計表       紙出力
    スタッフ        画面上で            タブレット          画面で        ・CSV
    現場            ドラッグ            で記入＋署名        金額編集      ・サイネージ
    車両            ＆ドロップ                              保存
    資格
```

### 主な機能カテゴリ

| カテゴリ | 主な機能 |
|---|---|
| **基盤データ** | 営業所／スタッフ／現場／車両／資格／ユーザーアカウント |
| **配置計画** | カレンダー（週・月表示）／一括配置／日別ステータス切替 |
| **現場報告** | 出来高確認書の作成・施主署名・印刷 |
| **支払い計算** | 日計表（スタッフ別の日次計算表）／累計残高の自動計算 |
| **印刷物** | 配置表（A3／A4横）／配置日報／日計表／出来高確認書 |
| **データ共有** | サイネージ（大画面モニター）／QRコードログイン／CSV出力 |
| **管理・運用** | 権限管理／監査ログ／QRコード発行 |

## 3. 用語集

本マニュアル全体で使う用語を、最初にまとめて定義します。

| 用語 | 意味 |
|---|---|
| **営業所** | 本社および各支店。例：「本社」「高瀬営業所」。スタッフと現場は必ず 1 つの営業所に所属します。 |
| **現場（サイト）** | 派遣先となる建設現場・案件。「現場名」と「元請け会社名」をセットで管理します。 |
| **スタッフ** | 現場に派遣する作業員。本システムでは管理対象の作業員全般を指します。 |
| **ユーザー** | システムにログインする人のアカウント。スタッフとは別概念です。1 人のスタッフに対して 1 つのユーザーを紐付けることができます。 |
| **配置（アサイン）** | 「いつ」「誰が」「どの現場に」行くかをカレンダー上に設定すること。期間（開始日〜終了日）で指定します。 |
| **配置日** | 配置の 1 日単位のレコード。1 つの配置が複数日にまたがる場合、日付ごとに「予定／休み／完了」を切り替えられます。 |
| **日勤／夜勤** | シフト区分。日勤の初期時間は 08:00〜18:00、夜勤は任意設定です。画面上は太陽アイコン（日勤）／月アイコン（夜勤）で区別されます。 |
| **通い／出張** | 配置区分。通いは毎日自宅から通う案件、出張は宿泊を伴う案件。配置日報では出張者が黄色ハイライトされます。 |
| **保険種別** | スタッフの保険区分。「社保」と「国保（一人親方）」の 2 種類。現場側に「社保のみ／国保のみ」の要件があれば、配置時に警告されます。 |
| **資格** | スタッフが保有する免許・技能講習修了などの総称。「普通自動車免許」「玉掛け」「フルハーネス特別教育」などがあります。 |
| **出来高確認書** | 現場で施主にサインをもらう実作業報告書。「株式会社マツケン」の正式様式で印刷できます。従来の紙伝票を置き換えるもの。 |
| **日計表** | スタッフ別に 1 日分の支払金額を計算する表。本日残・累計残が自動計算されます。 |
| **期首残高** | システム導入前からの前期繰越金額。スタッフごとに「いくら」「いつから集計するか」を設定しておくと、その日以降を累計残高に積み上げます。 |
| **サイネージ** | 各営業所に設置する大画面モニター用の表示画面。配置予定の確認専用で、触っても予定は書き換わりません。 |
| **QRログイン** | ユーザーごとに発行する QR コードをスマホで読み取ってログインする方法。パスワード入力が不要です。 |
| **監査ログ** | 誰がいつどのデータを変更したかの履歴。管理者のみ閲覧できます。 |

## 4. 利用者の種類と権限一覧

本システムは、ログインユーザーの「権限（ロール）」によって使える機能が変わります。ロールは 5 種類です。

| ロール | 想定する人 | できること（概要） |
|---|---|---|
| **管理者（admin）** | 本社の管理者・経営層 | すべての操作が可能。ユーザー管理・監査ログ閲覧も可。 |
| **所長（manager）** | 各営業所の所長 | 配置・スタッフ・現場・日計表・出来高確認書の編集。ユーザー管理・監査ログは不可。 |
| **事務（office）** | 各営業所の事務職 | 所長と同等（ユーザー管理・監査ログ・車両削除以外はすべて可）。 |
| **閲覧（viewer）** | 関係者・役員など閲覧のみ | カレンダー・スタッフ・現場一覧・出来高確認書の閲覧のみ。編集は不可。 |
| **スタッフ（staff）** | 現場作業員本人 | 自分の配置だけを閲覧。サイネージでの「見ました」確認、出来高確認書の閲覧のみ。 |

### 機能別アクセス可否（早見表）

| 機能 | 管理者 | 所長 | 事務 | 閲覧 | スタッフ |
|---|:-:|:-:|:-:|:-:|:-:|
| カレンダー閲覧 | ✅ | ✅ | ✅ | ✅ | 自分の分のみ |
| 配置の作成・編集・削除 | ✅ | ✅ | ✅ | ❌ | ❌ |
| 一括配置 | ✅ | ✅ | ✅ | ❌ | ❌ |
| スタッフ管理 | ✅ | ✅ | ✅ | 閲覧のみ | ❌ |
| 現場管理 | ✅ | ✅ | ✅ | 閲覧のみ | ❌ |
| 車両管理（追加・編集） | ✅ | ✅ | ✅ | ❌ | ❌ |
| 車両削除 | ✅ | ✅ | ❌ | ❌ | ❌ |
| 出来高確認書 | ✅ | ✅ | ✅ | 閲覧のみ | 閲覧のみ |
| 日計表 | ✅ | ✅ | ✅ | ❌ | ❌ |
| CSV出力 | ✅ | ✅ | ✅ | ❌ | ❌ |
| 印刷（配置日報） | ✅ | ✅ | ✅ | ❌ | ❌ |
| サイネージ | ✅ | ✅ | ✅ | — | 閲覧のみ（QR経由） |
| ユーザー管理 | ✅ | ❌ | ❌ | ❌ | ❌ |
| 監査ログ | ✅ | ❌ | ❌ | ❌ | ❌ |

> 注意：「所長」と「事務」の差は、ほぼ車両削除権限のみです。実運用では「所長 = 営業所の責任者」「事務 = 事務入力担当」のように使い分けることを想定しています。

# 第2部 ログインと基本操作

## 5. ログイン方法

### 5-1. ID・パスワードでログイン（PC）

1. ブラウザで本システムの URL（例：`https://matsken.example.com/login`）を開きます。
2. 左側に青いブランドパネル（「マツケン 配置管理システム」）、右側にログインフォームが表示されます。
3. 『ユーザー名』欄にログイン ID を入力します（例：`admin`）。
4. 『パスワード』欄にパスワードを入力します。右端の目のアイコンで表示／非表示を切り替えられます。
5. 『ログイン』ボタンをクリックします。
6. 成功すると、管理者・所長・事務・閲覧は『カレンダー』画面へ、スタッフは『サイネージ』画面へ自動で移動します。

**デモアカウント（初期設定）：**
- ユーザー名：`admin`
- パスワード：`demo1234`

> 注意：本番運用前に、必ず管理者のパスワードを変更してください。方法は「第3部 13. ユーザー管理」を参照。

### 5-2. QRコードでログイン（スマホ・タブレット）

スタッフがスマホで自分の配置を確認する際や、現場事務所のタブレットを共有で使う際に便利な方法です。

**事前準備（管理者が行う作業）：**
1. 管理者が『ユーザー管理』画面で対象ユーザーの『QRログインコード』を開き、QR コードを発行します。
2. 発行した QR コードを印刷して、本人に手渡しするか封入して配布します。

**スタッフが QR コードでログインする手順：**
1. スマホやタブレットのカメラで QR コードを読み取ります。
2. ブラウザが自動的に起動し、ログイン処理が実行されます。
3. ログイン成功後：
   - スタッフロールなら『サイネージ』画面（自分の配置が見える）に移動します。
   - その他ロールなら『カレンダー』画面に移動します。
4. ログインが有効な期間中（原則 1 週間）は、ブラウザを閉じても再度 QR を読む必要はありません。

### 5-3. ログインできない時

| 症状 | 原因と対処 |
|---|---|
| 「ユーザー名またはパスワードが正しくありません」と表示される | 入力ミス、または大文字／小文字の違いの可能性。もう一度正確に入力してください。それでもダメなら管理者にパスワード再設定を依頼。 |
| 「ログインに失敗しました。通信を確認してください。」 | ネットワーク接続を確認。社内 Wi-Fi・モバイル通信ともに不調の場合は他のサイトも開けるか試してください。 |
| QRコードを読み取ったら「QRが無効です」と出る | QR コードが既に無効化されているか、期限切れ／削除済み。管理者に再発行を依頼してください。 |
| ログインした後、画面がすぐ『ログイン画面』に戻る | セッション切れです。再度ログインしてください。 |

## 6. 画面の基本構成

本システムは PC とスマホの両方に対応しています。画面の構成が少し異なるので、それぞれ説明します。

### 6-1. PC（ブラウザ）で見た時

ログイン後の画面は、大きく 3 つに分かれます。

```
┌──────────────────────────────────────────────────────┐
│  マツケン                     ← サイドバー上部：ロゴと開閉ボタン │
│  配置管理システム                                          │
│ ────────────────────────────────────────────────────│
│ [📅 カレンダー]    │                                     │
│ [👥 スタッフ]      │                                     │
│ [🏢 現場]         │        本体エリア                     │
│ [🚚 車両管理]     │        （ここに各画面の中身が表示）         │
│ [📄 出来高確認書]  │                                     │
│ [🧾 日計表]       │                                     │
│ [⬇ CSV出力]      │                                     │
│ [🖨 印刷]         │                                     │
│ [🖥 サイネージ]    │                                     │
│ [🛡 ユーザー管理]  │  （管理者のみ）                         │
│ [📜 監査ログ]      │  （管理者のみ）                         │
│ ────────────────────────────────────────────────────│
│ [↪ ログアウト]     │                                     │
└──────────────────────────────────────────────────────┘
```

- **左：サイドバー**
  - メニュー項目が縦に並びます。クリックで画面が切り替わります。
  - 上部の矢印アイコン（┤ ├）でサイドバーを折りたたむ／展開できます。折りたたむとアイコンだけになり、本体エリアが広がります。
  - 『サイネージ』は別タブで開きます（他の項目は同じタブで開きます）。
  - 『ユーザー管理』『監査ログ』は管理者のみに表示されます。

- **中央〜右：本体エリア**
  - 選択中の画面が表示されます。
  - 上部に「パンくず」（ホーム > カレンダー など）とページタイトルがあり、画面階層が分かります。

- **右上：ページ単位のアクション**
  - 画面によって、『新規作成』『印刷』『保存』などのボタンが並びます。

### 6-2. スマホ（モバイル）で見た時

```
┌─────────────────┐
│ マツケン 配置管理  │  ← 上部バー（タイトルのみ）
├─────────────────┤
│                 │
│                 │
│   本体エリア       │
│                 │
│                 │
├─────────────────┤
│ 📅   👥   🏢   📄   ⋯ │  ← 下部タブ（5 つ）
│カレ  スタ  現場  出来  その他
│ンダ  ッフ       高     │
└─────────────────┘
```

- **上部バー**：タイトル「マツケン 配置管理」のみ。メニューはありません。
- **下部タブ**：よく使う 4 つのメニュー（カレンダー／スタッフ／現場／出来高）＋『その他』。
- **『その他』をタップ**すると、下から「車両管理」「CSV出力」「印刷」「ログアウト」のメニューが出てきます。
- スタッフロールでログインしている場合は『カレンダー』と『出来高』のみ表示されます。

> 💡ポイント：スマホでもすべての機能が使えますが、配置表（カレンダー）の操作は PC やタブレットの方が快適です。スタッフ個人の予定確認や、出来高確認書の施主署名はスマホ／タブレットが便利です。

## 7. ログアウト・セッションの有効期限

- **ログアウト方法**
  - PC：サイドバー下部の『ログアウト』ボタンをクリック。
  - スマホ：下部タブの『その他』→『ログアウト』をタップ。
- **自動ログアウト（セッション有効期限）**：ログインから約 1 週間。期限が切れると自動的にログイン画面に戻ります。
- **共有 PC を使う場合**：必ずログアウトしてからブラウザを閉じてください。

# 第3部 マスターデータの登録（最初にやること）

システム導入時や新しい営業所・スタッフ・現場・車両が増えた時に、まずこの「基盤データ」を登録します。一度登録すれば、毎日の配置作業では選択するだけで済みます。

**推奨の登録順序：**

1. 営業所（確認のみ。通常は最初から登録済み）
2. 資格マスター（確認のみ。通常は最初から登録済み）
3. スタッフ
4. 現場
5. 車両
6. ユーザー（スタッフがシステムにログインする場合）

## 8. 営業所の確認

営業所は初期セットアップ時に登録されます（画面からの新規作成ボタンは用意されていません）。現在登録されている営業所は、各種フォーム・フィルターのドロップダウンやボタンから確認できます。

**初期セットアップで登録される営業所（一例）：**

| 名称 | コード | 色の例 |
|---|---|---|
| 本社 | HQ | 紫など |
| 高瀬営業所 | TKS | 緑など |
| 草津営業所 | KST | 青など |
| 京都営業所 | KYT | オレンジなど |

- 各営業所には色が割り当てられ、スタッフ一覧・カレンダー・サイネージ等で視覚的に区別されます。
- 営業所の追加・名称変更・色変更が必要な場合は、導入担当者に連絡してください。

## 9. 資格マスターの確認

スタッフの保有資格をチェックボックスで選べるよう、資格マスターも初期セットアップ時に登録済みです。代表的な資格：

- 普通自動車免許
- 大型自動車免許
- フルハーネス特別教育
- 玉掛け技能講習
- 足場組立等作業主任者
- 職長・安全衛生責任者
- 高所作業車運転
- 酸素欠乏危険作業

資格の追加が必要な場合は、導入担当者に連絡してください。

## 10. スタッフ管理

**画面URL：** `/staff`（スタッフ一覧）、`/staff/new`（新規登録）、`/staff/[ID]`（編集）
**アクセス権限：** 管理者・所長・事務が編集可。閲覧は全ロール。

### 10-1. 一覧画面の見方

左サイドバーの『スタッフ』をクリックすると一覧画面が開きます。

**表示される列：**

| 列名 | 内容 | 備考 |
|---|---|---|
| コード | 社員コード（例：M001） | モノスペースで表示 |
| 氏名 | 名前 + フリガナ | フリガナはタブレット以上で表示 |
| 営業所 | 所属営業所（色付きバッジ） | 携帯では氏名の下に表示 |
| 保険 | 「社保」または「国保」 | |
| 資格 | 保有資格（最大 3 つ、超過分は「+N」表示） | 大画面のみ |
| 電話 | 電話番号 | タブレット以上 |
| （操作） | 『編集』ボタン | |

**フィルター・検索：**

- 上部の検索ボックスに名前／フリガナ／社員コードの一部を入力すると、即座に絞り込みされます。
- 営業所ボタン（『全て』『本社』『高瀬営業所』...）をクリックすると、その営業所のスタッフだけを表示します。
- 画面上部に「N 名表示中」というカウントが出ます。

### 10-2. 新規登録

1. 一覧画面の右上『スタッフ登録』（または『+ 新規作成』）をクリック。
2. フォームが表示されます。上部の「* 印がついた項目は必ず入力してください」に従って入力します。

**［基本情報］セクション**

| 項目 | 必須/任意 | 内容 | 例 |
|---|---|---|---|

### 10-4. 期首残高（openingBalance）の意味と設定タイミング

システム導入前から続いているスタッフへの前払い金や未精算分を、日計表の累計残に正しく反映するための仕組みです。

- **何に使う？**：「2026年4月1日にシステムを始めたが、そのスタッフには既に 30 万円を前払いしていた」という場合、日計表の累計残を正しく継承するために使います。
- **設定するもの 2 つ**：
  - **期首残高（円）**：導入時点での残高（例：300,000）。
  - **期首残高日（YYYY-MM-DD）**：集計を開始する日付（例：2026-04-01）。
- **どう使われる？**：その日付以降の「本日残」を積み上げて累計残を計算するときに、期首残を出発点として加算します。期首残高日より前の日計データは、累計に含まれません。
- **設定タイミング**：
  - システム導入時に一括で設定（管理者・導入担当者）。
  - 通常運用中は変更不要。
  - 前期繰越額が確定した年度始めに見直すのが一般的です。

> 注意：現状 UI のスタッフ編集フォームにはこの 2 項目の入力欄がありません。導入時には管理者とご相談の上、データとして設定する運用です。

- QR コードは他人に渡さないでください。
- 退職スタッフの QR コードは必ず無効化してください。
- 印刷して配布する場合は他人の目に触れないように封入してください。

### 14-1. スタッフ別ビュー／現場別ビュー／時間割ビュー

ツールバーの『表示モード』ボタンで切り替えます。

**スタッフ別ビュー（初期表示）**

- セルの中：配置された現場名（短縮表示）

- 縦軸：現場（現場名＋コード＋元請け）

- セルの中：配置されたスタッフ名（または人数）
- 用途：**「ある現場に誰が何人入っているか」を俯瞰する時に使う**。

**時間割ビュー（タイムテーブル）**

- 縦軸：時間帯（6:00〜21:00 を 30 分刻み）
- 横軸：スタッフ（選択日のみ）
- 用途：**「その日、誰が何時から何時まで働くか」を時間軸で見る**。夜勤・早朝シフトの把握に便利。
- 画面上部に日付選択欄があり、前日／翌日ボタンと「今日」ボタンで日付を移動できます。

- **4週間**：1ヶ月の全体像を見たい時に。

**日付ナビゲーション：**

- 『<』ボタン：前の期間へ移動（1週間なら前週、4週間なら前月）。
- 『>』ボタン：次の期間へ移動。
- 『今日』ボタン：本日を含む週／期間に戻る。

**キーボードショートカット：**

| キー | 動作 |
|---|---|
| ← | 前の期間へ |
| → | 次の期間へ |
| T | 今日の週に戻る |
| Esc | 開いているパネルを閉じる |

### 14-3. フィルター（スタッフ検索・営業所フィルター）

大きなマツケンでは 170 名以上のスタッフがいるため、絞り込みが必須になります。

**検索モード（テキスト検索）：**

- 検索ボックスに名前・フリガナ・社員コードの一部を入力。即座に絞り込まれます。
- スタッフ別ビューでは「スタッフを名前・コードで検索」。
- 現場別ビューでは「現場を名前・コードで検索」。

**選択モード（チェックボックス）：**

- 検索ボックス横の『選択』ボタン（チェックリストアイコン）をクリック。
- ドロップダウンに全スタッフ／全現場が一覧表示されます。
- チェックボックスで任意の複数を選択。選択件数が右下に表示されます。
- 『全選択』『全解除』のリンクあり。

**営業所フィルター：**

- ツールバー下のボタン列で、営業所を切り替え。

### 14-4. 祝日・日曜日・土曜日の表示

- **日曜日・祝日**：日付セルが赤色表示。祝日名（例：「成人の日」）も小さく表示されます。

## 15. 配置の新規作成（3通り）

### 15-1. セルをクリックして 1 件作成

もっとも単純な方法です。

1. カレンダーの空きセル（配置がない日）をクリック。
2. 右側に『配置追加』パネル（AssignmentPanel）がスライドして開きます。

   - **現場**（必須）：ドロップダウンで選択
   - **期間（開始日・終了日）**：セルの日付が自動入力されています。単日で良ければそのまま。
   - **区分**：通い／出張
   - **シフト**：日勤（太陽）／夜勤（月）
   - **時間**：開始時刻・終了時刻（日勤の既定は 08:00-18:00）
   - **車両**：（任意）
4. 『作成』ボタンで保存。

### 15-2. セルをドラッグして複数日を一括作成

期間を指定して配置したい場合に便利です。

1. 開始セル（例：4/10 月曜日）でマウスを押下（マウスダウン）。
2. そのまま横にドラッグ（4/11, 4/12, 4/13...）。
3. 終了セルで離す（マウスアップ）。
4. 右側に『配置追加』パネルが開き、開始日と終了日がセットされた状態で表示されます。
5. 現場、シフトなどを入力して『作成』。

> 💡ポイント：ドラッグ中は対象セルがハイライト表示されるため、期間を目視で確認しながら選択できます。

### 15-3. 一括配置（複数スタッフを同じ現場・期間に一度で配置）

「大型現場で 5 名を 2 週間派遣」のような場面に最適です。

1. ツールバーの『一括配置』ボタンをクリック → 選択モードに入ります。
2. スタッフ別ビューの左列で、配置したいスタッフにチェック。
3. 画面下にフローティングバー「N 名選択中」が表示されます。
4. 右側に『一括配置』パネル（BulkAssignmentPanel）が開きます。
5. 現場・期間・シフト・時間・車両を設定。
6. 『確定』ボタンで全員分の配置をまとめて作成。
7. 作成結果：
   - 成功：「N 名の配置を作成しました」。
   - 警告：スタッフごとに二重配置や保険要件の警告が出ます（詳細は 20 節）。

## 16. 配置パネルの入力項目（必須／任意）

『配置追加』『配置編集』パネル共通の項目です。

| 項目 | 必須/任意 | 内容 |
|---|---|---|
| スタッフ | 必須（現場ビューで新規作成時のみ選択） | 対象スタッフ。スタッフ別ビューから作成する場合は自動選択。 |
| 現場 | 必須 | 配置先の現場。ドロップダウン。状態「進行中」の現場のみ表示。 |
| 開始日 | 必須 | YYYY-MM-DD |
| 終了日 | 必須 | 開始日以降の日付 |
| 区分 | 必須 | **通い**（毎日帰宅）／**出張**（宿泊あり） |
| シフト | 必須 | **日勤**（太陽アイコン）／**夜勤**（月アイコン、紫色） |
| 開始時刻 | 必須 | 既定 08:00。24 時間制 HH:MM |
| 終了時刻 | 必須 | 既定 18:00 |
| 車両 | 任意 | ドロップダウン。「— なし —」で未指定も可 |
| 備考 | 任意 | 自由記述 |

## 17. 配置の編集・削除

**編集：**

1. カレンダー上の既存配置カードをクリック。
2. 右側に『配置編集』パネルが開きます。
3. 変更したい項目を修正。
4. 『更新』ボタン。

**削除：**

1. 同じく既存配置をクリックして編集パネルを開く。
2. 右上または下部のゴミ箱アイコンをクリック。
3. 確認ダイアログで『削除』。
4. カレンダーから該当配置が即座に消えます。

**右クリック（コンテキストメニュー）：**

- 配置カードを右クリックすると、メニューが表示されます：
  - 編集
  - 削除
  - **出来高確認書を新規作成**：この配置の日付・現場が自動入力された状態で『出来高確認書』新規作成画面に移動します。

## 18. 配置の移動（ドラッグ&ドロップで日付やスタッフを変更）

予定変更が入った時に便利です。

1. 既存の配置カードをドラッグ開始（マウスでクリックして離さずに動かす）。
2. 別のスタッフの行、または別の日付セルの上でドロップ。
3. 『配置を移動しますか？』確認ダイアログが開きます：
   - **現場**：（変更なし）
   - **移動元**：旧スタッフ／旧日付
   - **移動先**：新スタッフ／新日付
   - **日程変更**：日数差
4. 『移動する』で確定。
5. 移動元の配置は消え、移動先に新しい配置が作られます。

## 19. 日別ステータス管理（予定／休み／完了の切替、確認済みフラグ）

1 つの配置が「4/1〜4/14 の 2 週間」にわたる場合でも、途中で「4/7 は休み」のような日別変更が可能です。

**操作手順：**

1. 対象の配置カードをクリック → 『配置編集』パネルを開く。
2. パネル内の「日別管理」セクション（7 列のカレンダーグリッド）を確認。
3. 日別のマス目に以下が表示されます：
   - ◯ 予定（通常状態）
   - △ 休み
   - ✓ 完了
4. 日付マスをクリック：
   - 1 回クリック：予定 → 休み
   - もう 1 回：休み → 完了（または 予定に戻る）
5. 複数日をドラッグで一括切替も可能。
6. 画面下部に「出勤 N 日 / 休み N 日」のカウンタが表示されます。

**確認済みフラグ（スタッフ本人が使用）：**

- スタッフロールのユーザーがサイネージで自分の配置を見た時に、「見ました」ボタンを押すと、その配置日の「確認済み」フラグが立ちます。
- 誰がいつ確認したかは日別ステータスに記録されます。

## 20. 警告が出るケース

配置作成・更新時に、安全運用のための警告が出ることがあります。

### 20-1. 同じスタッフの二重配置

- **発生条件**：同じスタッフの同じ日に、すでに別の配置（予定状態）がある場合。
- **表示**：『配置追加』パネル下部に赤い警告メッセージ。競合する現場名と日付がリストされます。
- **対応**：
  - 正しい予定なら「それでも配置する」ボタンで強制作成。
  - 間違いなら『キャンセル』で戻り、既存配置を編集する。
- **カレンダー上の表示**：二重配置がある日付セルには赤い三角（AlertTriangle）アイコンが付きます。

### 20-2. 保険種別（社保／国保）が現場要件と合わない

- **前提**：現場マスターで「必須保険種別」を設定している場合（`any`／`company_only`／`national_only`）のみ警告。UI 上はこの設定項目は未表示のため、必要時は導入担当者にご相談ください。
- **発生条件**：例「社保のみ」現場に、国保スタッフを配置しようとした場合。
- **表示**：警告メッセージ「スタッフ：国保、現場要件：社保のみ、現場名：◯◯」。
- **対応**：運用に問題なければ「それでも配置する」。または別スタッフに変更。

### 20-3. 強制配置する方法

上記 2 つの警告（409エラー）が出た場合、パネル下部に『それでも配置する』（強制作成）ボタンが現れます。クリックすると警告を無視して作成します。

> 注意：警告を無視した配置は、あとで監査ログや支払計算で食い違いの原因になる可能性があります。強制配置する時は必ず理由を『備考』欄に記入してください。

# 第5部 サイネージ（大画面モニター表示）

**画面URL：** `/signage`
**アクセス権限：** 管理者・所長・事務・スタッフ（自分の分のみ）

## 21. サイネージとは

各営業所に設置する **タッチ対応の大型モニター（テレビ or 43〜55 型ディスプレイ想定）** に表示する、配置予定の確認用画面です。

- スタッフ（特にスマホを持たない作業員）が朝の出勤時に「今日どこに行くのか」を見るためのもの。
- 従来は紙で印刷したホワイトボードを運用していましたが、DX 化の肝になる機能。
- 操作はタップ（クリック）のみ。配置予定の書き換えはできません（書き換えは管理者が『カレンダー』画面で）。

## 22. 起動方法・URLパラメータ

**起動方法：**

- 左サイドバーの『サイネージ』をクリック（別タブで開きます）。
- 各営業所のモニター PC では、起動時に自動的にこの URL を開く設定を推奨。

**URL パラメータ：**

営業所や表示日数を URL で指定できます。

```
/signage?branch=HQ&days=10
```

| パラメータ | 意味 | 既定値 | 備考 |
|---|---|---|---|
| `branch` | 営業所コード（HQ、TKS、KST、KYT など） | （全営業所） | 営業所コードを指定すると、そのタブが初期選択されます |
| `days` | 表示日数 | 7 | 3〜14 の範囲。大画面では 7〜10 日が見やすい |

## 23. 画面の見方

### 23-1. 画面の全体構成

```
┌─────────────────────────────────────────────────────┐
│ 本社           配置予定           [個人を選ぶ] 16:42    │
│                                                 4/20(月)│
├─────────────────────────────────────────────────────┤
│ [本社] [高瀬] [草津] [京都]  ← 営業所タブ                  │
├─────────────────────────────────────────────────────┤
│  4/20(月)   │  4/21(火)   │  4/22(水)   │             │
│  ☀ 日勤3名  │  ☀ 日勤5名  │  ☀ 日勤4名  │             │
│  🌙 夜勤1名 │  🌙 夜勤0名 │  🌙 夜勤2名 │  ...        │
│  （クリックで詳細）                                       │
├─────────────────────────────────────────────────────┤
│                                         最終更新 16:42  │
└─────────────────────────────────────────────────────┘
```

- **上部**：営業所名、右側に大きなデジタル時計（HH:mm）と本日日付。
- **営業所タブ**：複数営業所がある場合、タブで切り替え。
- **日付カード**：表示期間の各日をカード表示。日勤・夜勤の人数を大きく表示。
- **フッター**：最終更新時刻（30 秒ごとに自動更新）。

### 23-2. 日付タップ → 当日の配置詳細モーダル

日付カードをタップ（クリック）すると、その日の詳細が画面いっぱいに表示されます。

**表示内容：**
- 日付・曜日・祝日名
- 日勤人数・夜勤人数の合計
- 現場ごとにグルーピングされたスタッフ一覧カード
  - 各スタッフ：名前、シフト種別（日勤／夜勤）、開始〜終了時刻
  - 現場の営業所カラーで色分け

背景の × ボタンまたは ESC で閉じます。

### 23-3. 「個人を選ぶ」から自分の予定を見る

スタッフが自分の今後の予定を確認するための機能です。

1. 画面右上の『個人を選ぶ』ボタンをタップ。
2. スタッフピッカーが開きます（2〜3 列のグリッドでスタッフ名が並ぶ）。
3. 自分の名前をタップ。
4. 画面が切り替わり、選択したスタッフの個人予定ビューになります：
   - 画面上部：自分の名前（大きく表示）と「個人予定」ラベル
   - 下部：期間内の全日付カード
     - 配置なしの日：「-」
     - 配置ありの日：元請け名／現場名／時間帯／シフト／『見ました』ボタン
5. 『全体に戻る』ボタンで元の営業所ビューに戻れます。

## 24. スタッフの「見ました」ボタン

自分の予定を確認したスタッフが、管理側に「確認済み」を伝える機能です。

1. 個人予定ビューで該当日付の配置カードを探す。
2. カード内の『見ました』ボタンをタップ。
3. ボタン色がグレー → 緑に変わり、確認日時が記録されます。
4. この記録は管理側のカレンダーや監査ログで確認できます。

> 💡ポイント：スタッフが確認ボタンを押したかどうかで、重要な予定変更が伝わっているかを把握できます。

## 25. 管理モード（管理者ログイン時のみ）

管理者・所長・事務のアカウントでログインしている場合、画面上部に小さな「管理モード」ラベルが表示されます。

- 管理モードでは、サイネージから直接配置の一部操作が可能になる場合があります（詳細は今後のアップデート予定）。
- 通常のスタッフ利用時には「管理モード」ラベルは出ません。

## 26. 自動更新

- 画面は 30 秒ごとに時計表示を更新します。
- 画面右下に「最終更新 HH:mm」が表示されます。
- 配置データそのものは、ブラウザを手動で再読み込みすると最新になります（自動更新は時計のみ）。

> 💡ポイント：配置変更を即時反映したい場合は、モニターを再読み込み（F5）してください。業務端末では「夜中に自動再読み込みする」設定を推奨します。

# 第6部 出来高確認書（現場報告フォーム）

**画面URL：** `/forms`（一覧）、`/forms/new`（新規作成）、`/forms/[ID]`（編集／閲覧）
**アクセス権限：** 管理者・所長・事務が編集可。閲覧は全ロール。

## 27. 出来高確認書とは

従来、現場で作成されていた紙の「出来高確認書」（通称：伝票、受領書）を、タブレット／スマホでデジタル入力し、施主に画面で直接サインをもらい、A4 で印刷・PDF 化できる機能です。

- サポスケにはなかった機能で、**本システムの肝**の 1 つ。
- マツケン社が 20 年以上使ってきた紙伝票の様式を、ほぼそのまま再現しています（赤い二重枠、令和年号、社印位置）。
- 施主の署名は **タッチ入力（指 or タッチペン）で直接画面にサイン** できます。
- 署名済みの確認書は**以降編集不可**（改ざん防止）、閲覧・印刷のみ可能。

## 28. 一覧画面（検索・プレビュー・削除）

左サイドバーの『出来高確認書』をクリック。

**表示される列：**

| 列名 | 内容 |
|---|---|
| 日付 | 工事日 |
| 現場 | 現場名（＋現場コード） |
| 作業内容 | 工事内容の抜粋（タブレット以上で表示） |
| 人数 | 従事したスタッフの人数（タブレット以上） |
| 出来高 | 数量＋単位（大画面のみ） |
| 状態 | 「下書き」「提出済み」のバッジ |
| 操作 | 『👁️（プレビュー）』『✎（編集）』『🗑️（削除）』 |

**検索：**

- 画面上部の検索ボックスに日付・現場名・現場コード・作業内容の一部を入れて絞り込み。

**操作：**

- **プレビュー（目アイコン）**：画面右側に確認書プレビューパネルが開きます。印刷前の見た目を確認できます。
- **編集（鉛筆アイコン）**：編集画面へ。
- **削除（ゴミ箱）**：確認ダイアログ → 『削除する』で削除。

**右上のアクション：**

- 『+ 新規作成』：新規作成画面へ。

## 29. 新規作成

1. 一覧画面右上の『新規作成』ボタン。
2. 確認書作成画面が開きます。上部にタブ「入力」「サイン」があり、画面の状態に応じて切り替わります。

### 29-1. 現場・日付・工事内容・数量・単位

画面上部の入力カードで、確認書の基本情報を入力します。

| 項目 | 必須/任意 | 内容 | 例 |
|---|---|---|---|
| 宛先（現場） | 必須 | 現場を選択 | `スリーエフコーポレーション` |
| 日付 | 必須 | 工事を行った日付 | `2026-04-20` |
| 作業内容 | 任意 | 実施した作業内容 | `雑工、足場組立等` |
| 数量 | 任意 | 出来高の数量 | `49` |
| 単位 | 任意 | 単位（㎡ / 棟 / 日 など） | `m²` |
| 残業 (h) | 任意 | 残業時間（時間単位、0.5 刻み可） | `1.5` |
| 始業 | 任意 | 開始時刻 | `08:00` |
| 終業 | 任意 | 終了時刻 | `17:00` |

### 29-2. 従事スタッフ

画面の中央セクションで従事したスタッフを入力します。

1. デフォルトで 1 行空欄の入力行が表示されます。
2. 各行：
   - **氏名**：作業者の名前。
   - **保険種別**：「社保」または「一人親方」（= 国保）。
3. **『+ 追加』ボタン**で行を追加。
4. **『×』ボタン**で行を削除（2 行以上ある時のみ）。
5. **『カレンダーから読込』ボタン**：現場と日付を選択済みなら、その日その現場に配置されているスタッフ名と時間帯を自動取り込みできます。取り込み後も個別に編集可能。

> 💡ポイント：『カレンダーから読込』を使うと、配置画面で登録済みの情報がそのまま流し込まれるため入力が一気に楽になります。

### 29-3. 施主署名（手書き／タッチ・マウス対応）

入力が完了したら、画面下の『入力完了 → サインへ』ボタンまたは上部タブの『サイン』をクリック。

1. サインモードの画面に切り替わります。プレビューとして出来高確認書の簡易表示が上部に出ます：
   - 『令和◯年◯月◯日』（和暦）
   - 『宛先：㈱〇〇様』
   - 『現場名』『作業内容』『数量 + 単位』『時間』『作業者一覧』
2. 下部に広い署名欄があります。タブレットなら指／タッチペンで、PC ならマウスで署名を書きます。
3. 描き直したい場合は『クリア』ボタンでリセット。

### 29-4. 下書き保存と「提出済み」フラグ

- **下書き保存**：サイン前でも『保存』ボタンで途中保存できます。状態は「下書き」のまま。
- **署名済み＝提出済み**：施主署名が入った状態で保存すると、自動的に「提出済み」フラグが立ちます。
- **『保存して印刷』ボタン**：署名後、このボタンで保存＋ブラウザの印刷ダイアログが開きます。
- **提出済みの書類は編集不可**：間違いに気づいたら、該当書類を削除して新規作成し直してください。

## 30. 印刷（A4縦、赤い二重枠、令和年号付き公式フォーマット）

**印刷用レイアウト：**

- A4 縦向き（210mm × 297mm）
- 赤い二重線枠で囲まれたマツケン公式フォーマット
- 会社情報ヘッダー：
  - 株式会社マツケン
  - 〒571-0046 大阪府門真市本町21番21号
  - TEL (06) 6903-3200 / FAX (06) 6903-3296
- 左上：タイトル「出来高確認書控」「（経理用）」
- 本文テーブル：宛先・日付（令和年号）・現場名・作業内容・作業者リスト・数量・単価・金額（単価と金額は空欄、現場側で手書き想定）・始業・終業・残業・サイン欄

**印刷手順：**

1. 編集画面または閲覧画面で『印刷』ボタンをクリック。
2. ブラウザの印刷ダイアログが開きます。
3. プリンタ選択して印刷。

**FAX 送信に対応：**

- 印刷物を各営業所や施主に FAX 送信する運用にも対応した見た目になっています。
- 令和年号・赤枠・会社印位置など、既存の紙様式と揃えてあります。

## 31. カレンダーから直接作成する導線

配置情報を基に確認書を作る場合、カレンダーからワンクリックで新規作成画面を開けます。

1. カレンダー画面で該当の配置カードを**右クリック**。
2. コンテキストメニューから『出来高確認書を新規作成』を選択。
3. 新規作成画面が開き、日付と現場が自動入力された状態で始まります。
4. あとは作業内容・数量などを入力してサインをもらうだけ。

# 第7部 日計表（日次支払計算）

**画面URL：** `/tally`（編集画面）、`/print/daily-tally`（印刷画面）
**アクセス権限：** 管理者・所長・事務

## 32. 日計表とは

スタッフ別に「その日にいくら稼いで、いくら引かれて、累計残はいくらか」を計算する表です。毎日朝イチで日計データを締め、月末・月初に支払いをまとめる運用を想定しています。

**特徴：**

- 前日のカレンダー配置から、**その日の下書きが自動生成**されます（初めてその日の画面を開いた時）。
- 画面上でセルをクリックして**金額を直接編集**できます（スプレッドシートのような感覚）。
- **本日残（当日残）・累計残**は金額編集に応じてリアルタイムに自動計算・表示されます。
- 最大 **2 現場まで同じ日に記録可能**（掛け持ち対応）。

## 33. 画面の見方

### 33-1. 日付選択・営業所切替

左サイドバーの『日計表』をクリック。

**画面上部のツールバー：**

```
[◀] [2026-04-20] [▶] [今日]  [全営業所] [本社] [高瀬] [草津] [京都]
                                                  [保存(N件)] [印刷]
```

- **前日（◀）／翌日（▶）**：日付を 1 日ずつ移動。
- **日付入力**：直接入力でジャンプ。
- **今日**：本日に戻る。
- **日付ラベル**：選択中の日付が「2026年4月20日 (月)」と日本語表記で表示されます。
- **営業所タブ**：表示対象の営業所を切り替え。「全営業所」なら全スタッフを表示。
- **保存ボタン**：未保存の変更件数がバッジで表示されます。
- **印刷ボタン**：印刷画面（`/print/daily-tally`）へ遷移。

**サマリーバー：**

- 対象人数
- 本日の当日残合計（選択スタッフ全員分）
- 未保存の変更件数（あれば）

### 33-2〜4. テーブルの列構成

画面中央のテーブルは、左から右へ以下の列があります：

**1. スタッフ情報**
| 列 | 内容 |
|---|---|
| コード | 社員コード（モノスペース表示） |
| 氏名 | 営業所カラードット＋氏名 |
| 現場 | 配置された現場名（自動表示） |

**2. 支払明細（7 列 × 最大 2 現場）**

| 列 | 説明 | 初期値 |
|---|---|---|
| 基本 | 基本料金（日当） | スタッフの基本日当 |
| 運転 | 現場運転手当 | 0 |
| 自社 | 自社計画手当（社内用事の待機代など） | 0 |
| リフト | フォークリフト・高所作業車手当 | 0 |
| 特殊 | 特殊技能手当 | 0 |
| 他 | その他手当 | 0 |
| 追加 | 追加料金 | 0 |

- 掛け持ちで 2 現場ある場合、2 行（rowSpan）で表示されます。
- 各セルをクリックして数値を直接入力できます（Enter キーで次のセルへ）。

**3. 支払合計**
- 支払明細 7 列×最大 2 現場の合計。自動計算・リアルタイム更新。

**4. 相殺明細（4 列）**

| 列 | 説明 | 初期値 |
|---|---|---|
| 安全 | 安全会費 | 500 円（配置ある日は自動計上） |
| 宿泊 | 宿泊費 | 0 |
| 他 | その他相殺 | 0 |
| 前渡金 | 既払い前渡金 | 0 |

**5. 当日残・累計残**

- **当日残**：支払合計 − 相殺合計。自動計算。
- **累計残**：前日末の累計残＋当日残。自動計算。

### 33-5. 本日残・累計残の自動計算

以下の計算式で画面上に即時反映されます：

```
支払合計 = 現場1合計 + 現場2合計
相殺合計 = 安全会費 + 宿泊費 + その他相殺 + 前渡金
当日残 = 支払合計 − 相殺合計
累計残 = 期首残高 + Σ(openingBalanceDate 以降の当日残)
```

- **期首残高**（スタッフマスターで設定）がある場合、そこから積み上げ。
- **期首残高日**（設定日）より前のデータは累計に含みません（導入前ツケとの分離）。

## 34. 自動初期化のしくみ

**初めてその日の日計表を開いた時：**

1. その日に「予定」状態の配置（AssignmentDay.status = scheduled）をスタッフごとに集約。
2. 各スタッフに最大 2 現場まで紐付け：
   - 1 つ目 → 現場1
   - 2 つ目 → 現場2
   - 3 つ目以上 → 備考欄に「3件以上の配置あり（+N件は未反映）」と警告メモ
3. 基本料金（Site1BaseFee）に以下の優先順位で単価を設定：
   - その日の単価オーバーライド → 配置の単価オーバーライド → スタッフの基本日当 → 0
4. 安全会費を一律 500 円で自動計上。

**2 回目以降：**

- すでに保存済みのデータは上書きされません（ユーザー編集を尊重）。
- 配置を追加・変更しても、日計表の再初期化は自動では行われません。必要に応じて手動で該当セルを更新してください。

## 35. 編集と保存（インライン編集・Enter確定・一括保存）

**編集方法：**

1. 対象セル（例：現場1の基本料金）をクリック → 数値入力モードに。
2. 金額を入力（半角数字）。
3. **Enter キー**で確定し、次のセルへ。
4. 変更されたセルは背景色でマークされ、画面上部に「未保存の変更 N 件」と表示されます。
5. 『保存（N件）』ボタンをクリック → データベースに一括保存。

**特記事項：**

- 保存せずに日付を変えたり画面を離れたりすると、変更は失われます。
- ブラウザの「戻る」「閉じる」時にも警告は出ない仕様のため、必ず『保存』してから移動してください。

> 💡ポイント：金額は 3 桁区切り（カンマ）で表示されますが、入力は単純な数値でOKです（例：`17500`）。

## 36. 3件以上配置がある日の警告

- 同じスタッフが同じ日に 3 件以上の現場に配置されている場合、日計表には最大 2 件までしか反映されません。
- 3 件目以降は、備考欄に「3件以上の配置あり（+N件は未反映）」と自動でメモされます。
- 対応：配置を整理（不要な配置を削除）するか、2 現場分に金額を集約して入力してください。

## 37. 印刷

日計表画面右上の『印刷』ボタンをクリック → `/print/daily-tally` 画面へ移動。

**印刷フォーマット：**

- **用紙サイズ**：A3 横 or A4 横を選択可能。
- **タイトル**：「日計表」＋ 令和年月日表示
- **ヘッダー**：会社名、総出動数、営業所別内訳、出力日時
- **表組み**：画面と同じ構成（スタッフ・現場・支払明細・相殺明細・当日残・累計残）
- **行ハイライト**：支払合計＆相殺合計の列は薄色背景、計算誤差があれば赤字
- **複数営業所選択時**：営業所ごとにページ分割

**印刷手順：**

1. 日付・営業所を指定して『印刷』ボタン。
2. プレビュー画面で用紙サイズを選択。
3. ブラウザの印刷ダイアログで印刷実行。

## 38. 期首残高（openingBalance）の運用ルール

- **いつ設定するか**：システム導入時に、管理者が一括で設定。年度途中の変更は非推奨。
- **期首残高日をいつに設定するか**：システム運用開始日、または会計年度の初日に合わせる。
- **運用ルール例**：
  - 新年度 4/1 にリセット：`openingBalance = 0`、`openingBalanceDate = 2026-04-01`
  - 過去の前払い分を引き継ぐ：`openingBalance = 300000`、`openingBalanceDate = 2026-04-01`
- **システムへの反映**：スタッフマスターの 2 項目で保存。現状 UI フォームには未表示のため、導入担当者にご依頼ください。

# 第8部 印刷機能

本システムで印刷できるものは主に以下の 4 種類です。

| 印刷対象 | 用紙 | 主な用途 |
|---|---|---|
| 配置カレンダー（スタッフ別／現場別） | A3 横 or A4 横 | ホワイトボード代替。2週間〜3ヶ月分を一覧 |
| 配置日報（その日の配置一覧） | A4 横 | 朝礼用。その日の現場・人員の詳細 |
| 日計表 | A3 横 or A4 横 | 月末精算用。スタッフ別の支払計算明細 |
| 出来高確認書 | A4 縦 | 施主に渡す／FAX 送信する工事報告書 |
| QR コード | A4 | スタッフ個人のログインコード |

> 💡ポイント：本システムはスマホを持たないスタッフがいることを前提に設計されており、**紙での印刷物を FAX で各営業所に送る運用**にも対応しています。

## 39. 配置カレンダー印刷（スタッフ別／現場別）

**画面URL：** `/print/daily`（カレンダー画面の『印刷』ボタン経由）
**アクセス権限：** 管理者・所長・事務

### 39-1. 印刷開始方法

カレンダー画面から：

1. カレンダー画面右上の『印刷』ボタンをクリック（プリンタアイコン）。
2. 小さなポップオーバー『印刷範囲を選んでください』が開きます。
3. 開始日・終了日を選択。
4. 『印刷プレビューへ』をクリック。
5. 印刷プレビュー画面に遷移します。

### 39-2. 期間指定と用紙サイズ（A3横／A4横）

**用紙サイズ選択（プレビュー画面上部）：**

- **A3 横** — 用紙幅 約 1498px（12mm マージン後）。最大 90 日（3ヶ月）対応。「A3横（3ヶ月対応）」
- **A4 横** — 用紙幅 約 1050px。2週間〜1ヶ月向き。「A4横（2週間〜1ヶ月向き）」

### 39-3. プリセット（2週間／4週間／2ヶ月／3ヶ月）

プレビュー画面上部のプリセットボタンで、期間を自動設定できます。

| プリセット | 動作 |
|---|---|
| **2週間** | 開始日を月の 1・5・10・15・20・25 日にスナップして 14 日分表示 |
| **4週間** | 同スナップで 28 日分 |
| **2ヶ月** | その月の 1 日から 2 ヶ月分 |
| **3ヶ月** | その月の 1 日から 3 ヶ月分 |

> 💡ポイント：マツケン社の従来の紙様式は「1・5・10・15・20・25 日始まりの 2 週間サイクル」です。『2週間』プリセットがこれに完全対応します。

### 39-4. 日数に応じた文字サイズ自動調整

表示する日数が多いほどセルが狭くなるため、文字サイズは自動的に調整されます。

| 密度 | 1 日あたり幅 | フォント | 表示内容 |
|---|---|---|---|
| ゆとり (loose) | 36px 以上 | 9px | 曜日＋祝日名 |
| 通常 (normal) | 22-36px | 9px | 曜日＋祝日短縮 |
| 詰め (tight) | 14-22px | 8px | 曜日のみ |
| 超詰め (dense) | 14px 未満 | 7px | 最小表示 |

### 39-5. 表示モード（スタッフ別／現場別）

プレビュー画面のトグルで切り替えます。

**スタッフ別ビュー：**

- 行：スタッフ（社員コード・氏名）
- 列：日付
- セル：配置された現場（短縮表示）
  - 上段（黄色背景）：日勤
  - 下段（紫色背景）：夜勤

**現場別ビュー：**

- 行：現場（元請け・現場名・現場コード）
- 列：日付
- 表示オプションのトグル：
  - **人数のみ**（コンパクト）：セルに配置人数だけ表示。
  - **氏名あり**（詳細）：スタッフ名も表示。

### 39-6. 「会社名2文字＋現場名2文字」の短縮ルール

カレンダーのセル幅を節約するために、配置セル内の表示は以下のルールで短縮されます：

- スタッフ別ビューのセル（1 件の配置）：
  - 上段：**会社名（元請け）の先頭 2 文字**
  - 下段：**現場名の先頭 2 文字**
  - 例：元請け「スリーエフコーポレーション」、現場「本町ビル」→「スリ／本町」

> 💡ポイント：この短縮ルールはマツケン社が従来使ってきたエクセル手作業配置表と同じルールです。覚えてきた略記がそのまま使えます。

### 39-7. 色分け

- **日曜日・祝日**：赤系背景
- **土曜日**：青系背景
- **平日**：グレー系
- **日勤**：黄色
- **夜勤**：紫

### 39-8. ヘッダー・凡例・脚注

- **ヘッダー**：表示期間（例：2026-04-10 〜 2026-06-09）、日数（61日間）、用紙サイズ、出力日時、会社名「株式会社マツケン 配置管理システム」
- **凡例（スタッフ別）**：「上段=会社名、下段=現場名（各 2 文字）」
- **凡例（現場別）**：「上段=日勤、下段=夜勤」

# 第10部 管理・運用

## 47. 監査ログ（いつ誰が何を変更したか）

**画面URL：** `/audit-logs`
**アクセス権限：** 管理者のみ

「誰がいつ、どのデータをどう変更したか」を自動的に記録する機能です。誤操作・不正操作の追跡、データトラブルの原因究明に使います。

### 47-1. フィルター

左サイドバー『監査ログ』をクリック（管理者のみ表示）。

画面上部に 3 つのフィルターがあります。

| フィルター | 選択肢 |
|---|---|
| **対象（モデル）** | 配置／配置日／スタッフ／現場／ユーザー／出来高報告／車両 |
| **ユーザー** | 過去にログを残した全ユーザー名（最大 50 件） |
| **操作** | 作成／更新／削除 |

- いずれも未選択なら「すべて」を対象。
- 複数組み合わせて絞り込み可能。

### 47-2. 表示される列

| 列 | 内容 |
|---|---|
| 日時 | いつ変更されたか（日本時間、年月日時分秒） |
| ユーザー | 変更を行った人のユーザー名（削除済みなら「(システム)」） |
| 操作 | 作成／更新／削除（色付きバッジ：作成=青、更新=グレー、削除=赤） |
| 対象 | 配置・スタッフ・現場など |
| ID | 対象データの ID |
| 差分（概要） | 変更内容の概要（120 文字まで） |

### 47-3. 差分の詳細表示

- 差分セルをクリックすると、『▶』マークが展開されて JSON 形式で変更内容の前後比較が見られます。
- 例：「dailyRate を 15000 → 16000 に変更」

### 47-4. 表示件数と運用

- 最大 **直近 200 件** まで表示。
- 古いログはデータベースに残り続けますが、画面からは 200 件を超える閲覧はできません。
- 大きなトラブル調査時は導入担当者に依頼してください（SQL で抽出可能）。

## 48. データの定期バックアップについて

本システムのデータは SQLite ファイル（`dev.db` 相当）に保存されます。定期的なバックアップ計画を推奨します。

**推奨：**
- 週 1 回、データベースファイルを別メディア（クラウドストレージ等）にコピー。
- 月 1 回の CSV 出力を保管（配置履歴のアーカイブ）。
- 出来高確認書は PDF 化して経理フォルダに保存。

具体的なバックアップ運用は導入担当者と相談してください。

## 49. QRコード運用ルール

QR ログインは便利ですが、物理的な QR コード＝パスワードと同じです。以下のルールを守ってください。

- **発行時は本人に直接渡す**：封筒に入れて本人に封入渡し。
- **他人の目に触れる場所に掲示しない**。
- **紛失報告が入ったら即座に『再発行』または『無効化』**。
- **退職者の QR は必ず『無効化』**。
- **印刷した紙は本人が保管**。破棄する時はシュレッダー推奨。

## 50. パスワード運用ルール

- **管理者の初期パスワード（`demo1234`）は本番運用前に必ず変更**。
- **スタッフロールはパスワードよりも QR ログインを推奨**（入力の手間を削減）。
- **パスワードは 4 文字以上必須**。実運用では 8 文字以上推奨。
- **使い回し厳禁**：他のシステムとは別のパスワードを使用。
- **忘れた時は管理者が『ユーザー管理』画面からリセット（新しいパスワードを設定）**。

# 第11部 ロール別クイックガイド

## 51. 本社管理者（admin）向け 1日の流れ

**朝（9:00 頃）：**

1. PC でシステムにログイン。
2. 『カレンダー』で今日の配置状況を確認。ダブルブッキング警告（赤三角）がないかチェック。
3. 各営業所のサイネージが正しく表示されているか確認（各営業所に電話 or リモートで）。

**午前中：**

4. 『出来高確認書』一覧で、前日提出されていない未サインの書類がないかチェック。
5. 必要に応じて各営業所所長に連絡。

**午後：**

6. 翌日〜翌週の配置を『カレンダー』で確定。
7. 必要に応じて『印刷』→『配置日報』で翌日の配置表を印刷、FAX で各営業所へ送付。

**月末：**

8. 『日計表』で月末締めの支払計算をチェック。
9. 『CSV出力』で配置データを出力し、会計ソフトに取り込み。
10. 『監査ログ』でその月の大きな変更がないか確認。

## 52. 営業所長（manager）向け 1日の流れ

**朝：**

1. ログイン後、『カレンダー』を開き、自営業所のスタッフだけに絞り込み。
2. 当日の配置に問題がないか目視確認。
3. 営業所のサイネージが表示されているか確認（大画面モニター前でスタッフが見ている状態にする）。

**日中：**

4. 現場からの変更依頼（休み／振替／早退など）があったら、『カレンダー』から配置編集。
5. 『日別ステータス管理』で該当日を「休み」に。

**夕方：**

6. 現場から上がってきた出来高確認書（タブレット入力済み）を『出来高確認書』一覧で確認。
7. 翌日の配置に追加・変更があれば修正。

## 53. 事務（office）向け 1日の流れ

**朝：**

1. ログイン後、『日計表』で前日分を最終チェック・保存。

**日中：**

2. 出来高確認書が上がってきたら『日計表』で金額を反映。
3. 掛け持ち案件の金額集約を確認。
4. 必要に応じて『日計表』を印刷して、所長に確認してもらう。

**月末：**

5. 月次集計：『CSV出力』で月間配置データを出力。
6. 『印刷』→『日計表印刷』で営業所別の月末書類を印刷。

## 54. スタッフ（staff）向け 利用方法

**予定の確認：**

- スマホで配布された QR コードをカメラで読み取り → サイネージ画面が開く。
- または営業所のサイネージ（大画面モニター）で『個人を選ぶ』→ 自分の名前をタップ。
- 自分の今後の予定（当日〜7〜14 日先まで）がカード表示される。

**配置の確認通知：**

- 配置カードで『見ました』ボタンをタップ → 管理者に「確認済み」が伝わります。
- 重要な変更があった時は必ずタップしてください。

**出来高確認書の閲覧：**

- 過去に自分が関わった出来高確認書を閲覧のみ可能（編集はできません）。

**やってはいけないこと：**

- QR コードを他人に見せる・渡す・写真送付
- パスワードを共有する
- システムの設定を勝手に変更（そもそもできない権限ですが）

## 55. 閲覧（viewer）向け 利用方法

- 経営層の参考や、外部監査用の権限です。
- ログイン後、『カレンダー』『スタッフ』『現場』『出来高確認書』を閲覧できます。
- 編集・削除・作成は一切できません。
- サイネージ・ユーザー管理・監査ログは表示されません。

# 第12部 よくある質問・トラブル対応

## 56. ログインできない

| 症状 | 対処 |
|---|---|
| パスワードを忘れた | 管理者に連絡 → 『ユーザー管理』画面からパスワード再設定（新しいパスワードを通知してもらう） |
| ユーザー名が分からない | 管理者に確認 |
| QR が読み取れない | カメラレンズを拭く。QR コードの印刷が薄い場合は再発行を依頼 |
| QR を読んだが「無効です」と出る | QR が無効化されている可能性。管理者に再発行を依頼 |
| ログイン画面に戻される（セッション切れ） | 再度ログインしてください（1 週間の自動ログアウト） |
| 「通信を確認してください」エラー | ネットワーク接続を確認。社内 LAN が止まっていないか、Wi-Fi が届いているか |

## 57. カレンダーが真っ白／配置が表示されない

| 症状 | 対処 |
|---|---|
| カレンダー全体が空欄 | 画面上部の営業所フィルターで絞り込みすぎていないか確認。『リセット』で全営業所表示 |
| 一部のスタッフが見えない | スタッフ検索ボックスに文字が残っていないか確認 |
| 配置はあるはずなのに出ない | ブラウザを再読み込み（F5 キー）。それでもダメなら『今日』ボタンで今日に戻り、もう一度該当週へ移動 |
| 「読み込み中...」から動かない | ネットワークが不安定。ブラウザを再読み込み |

## 58. 配置作成時に警告が出た

| 警告 | 意味と対処 |
|---|---|
| 「◯◯さんはすでに ◯現場に配置されています」 | 二重配置警告。既存の配置を確認し、必要なら変更。問題なければ『それでも配置する』 |
| 「スタッフ：国保、現場要件：社保のみ、現場名：◯◯」 | 保険要件の不一致。別のスタッフに変えるか、現場側の要件を見直す |
| 「スタッフを選択してください」 | 現場別ビューで新規作成時、スタッフを未選択。左側のリストから選ぶ |
| 「現場と日付を選択してください」 | 必須項目未入力。フォームに戻って入力 |

## 59. 日計表の金額がおかしい

| 症状 | 対処 |
|---|---|
| 基本料金が 0 円になっている | スタッフの基本日当が未設定。『スタッフ』画面で日当を設定 |
| 累計残が前日と合わない | 『期首残高日』の設定を確認。その日より前のデータは累計に含まれない |
| 安全会費が 500 円以外になっている | 誰かが手動変更した可能性。監査ログで変更履歴を確認 |
| 3 件目の現場が反映されない | 仕様。2 現場までの表示。備考欄のメモを確認して手動調整 |

## 60. 印刷で文字が小さすぎる／見切れる

| 症状 | 対処 |
|---|---|
| 配置カレンダー印刷で文字が潰れる | 用紙サイズを A4 横から A3 横に変更。または期間を短縮（4 週間 → 2 週間など） |
| 日報が複数ページに分かれる | 配置人数が多い日は自然と複数ページになります。必要ならプリンタ設定で「ページに合わせる」を選択 |
| 出来高確認書の枠が欠ける | ブラウザの印刷ダイアログで「背景色・画像の印刷」を ON。余白設定を「最小」に |
| プレビューと印刷結果が違う | ブラウザにより微妙に挙動が異なります。Chrome または Edge を推奨 |

## 61. QRコードが使えない

| 症状 | 対処 |
|---|---|
| カメラアプリで読んでも何も起きない | QR がピントが合う距離で読み取る（20〜30cm 推奨） |
| ブラウザが開くが、ログイン画面に戻される | QR が無効化されている。管理者に再発行依頼 |
| スマホが変わって QR がスマホに入っていない | 新しいスマホのカメラで印刷済み QR を再スキャン、または管理者に紙の QR を再発行してもらう |

# 付録C 入力ルールまとめ

## 日付・時間

| 項目 | フォーマット | 例 |
|---|---|---|
| 日付 | YYYY-MM-DD | `2026-04-20` |
| 時間 | HH:MM（24 時間制） | `08:00`、`18:30` |
| 期間 | 開始日 ≤ 終了日 | 開始日を終了日より後にすることはできません |

## 必須項目

### スタッフ登録

- 社員コード（ユニーク）
- 営業所
- 氏名
- フリガナ
- 保険種別

### 現場登録

- 現場コード（ユニーク）
- 担当営業所
- 現場名

### 車両登録

- 車両ナンバー（ユニーク、50 文字以内）

### 配置作成

- スタッフ
- 現場
- 開始日 / 終了日
- 区分（通い／出張）
- シフト（日勤／夜勤）
- 開始時刻 / 終了時刻

### 出来高確認書

- 現場
- 日付

### ユーザー作成

- ユーザー名（ユニーク、1 文字以上）
- パスワード（4 文字以上 200 文字以内、新規時のみ必須）
- 表示名
- 権限

## 金額入力

- すべて整数（円単位、小数不可）
- マイナス値も可（前渡金の返却など特殊ケース）
- カンマは自動で表示、入力は `17500` のように数字のみで OK

## 数量・単位

- 出来高確認書の「数量」は文字列扱い（例：`49`、`30.5`、`3棟`）
- 単位は自由記述（`m²`、`棟`、`日`、`箇所`など）

## 残業時間

- 時間単位、小数可（0.5 刻み推奨）
- 例：`1`、`1.5`、`2`
